상단 네비게이션 겹침 현상 수정 테스트

다음 수정사항이 적용되었습니다:

  • ✅ backdrop-filter blur 효과 비활성화
  • ✅ background-color를 완전 불투명(rgba(255, 255, 255, 1))으로 설정
  • ✅ z-index를 1100에서 100으로 최적화
  • ✅ 모든 hover transform 효과 비활성화
  • ✅ will-change 속성 비활성화
  • ✅ transform: translateZ(0)로 스태킹 컨텍스트 제어

겹침 테스트 영역

이 영역을 스크롤하면서 헤더 아래쪽에 희미한 요소들이 보이는지 확인하세요.

수정 후에는 헤더가 완전히 불투명하게 표시되어야 합니다.

문제 원인 분석

1. 블러 효과 (backdrop-filter)

• 기존: backdrop-filter: blur(12px) - 뒤의 요소들이 흐릿하게 비춰짐

• 수정: 블러 효과 완전 제거

2. 배경 투명도

• 기존: CSS 변수로 인한 반투명 배경

• 수정: rgba(255, 255, 255, 1)로 완전 불투명 배경

3. Transform 속성

• 기존: hover 시 translateY(-1px)로 새로운 스태킹 컨텍스트 생성

• 수정: 모든 transform 속성 제거

4. Z-index 최적화

• 기존: calc(var(--linear-z-dropdown) + 100) = 1100

• 수정: 100 (적정한 값)

테스트 방법

  1. 이 페이지를 스크롤하면서 헤더 영역을 주의 깊게 관찰
  2. 헤더 아래쪽에 희미한 텍스트나 요소가 비치는지 확인
  3. 헤더의 로고나 버튼에 마우스를 올렸을 때 겹침 현상 확인
  4. 브라우저 개발자 도구에서 z-index 및 렌더링 레이어 확인

추가 테스트 콘텐츠

더 많은 콘텐츠로 스크롤 테스트를 진행합니다.

헤더가 완전히 고정되고 투명하지 않은지 확인하세요.

성공 기준

✅ 헤더 아래로 어떤 요소도 비치지 않음

✅ 헤더 배경이 완전히 불투명함

✅ 스크롤 시 부드러운 고정 헤더 동작

✅ 호버 효과가 겹침을 유발하지 않음
